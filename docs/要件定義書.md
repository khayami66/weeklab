## 2. プロジェクト概要

### 2.1 背景
小学校の理科専科教員は、毎週の週案作成にあたり、年間指導計画・時間割・クラス別進度・単元内容などを複数の情報源から手作業で突合している。特に理科専科は複数クラス（MVP想定では7クラス）を掛け持ちするため、クラスごとに進度が異なり、週案作成の負担が大きい。また、作成した週案は管理職に提出する必要があり、所定様式での印刷が求められる。

### 2.2 目的
以下の3点を達成する：
1. 週案作成の時間を大幅に短縮する
2. 進度管理の正確性を担保する
3. 管理職提出用に印刷可能な週案を自動生成する

### 2.3 対象範囲

| 層 | MVP | 将来 |
|---|---|---|
| 学校 | 松戸市立梨香台小学校 | 啓林館採用校（松戸市内→全国） |
| 教科 | 理科 | 国算社 他 |
| 学年 | 3年・4年 | 全学年 |
| 教科書 | 啓林館 | 東京書籍・大日本図書・光村 等 |
| ユーザー | 理科専科教員（本人） | 担任・他専科教員 |

### 2.4 配信・運用形態
- **Webアプリケーション**として Vercel 上にデプロイ
- URL を知る個人のみアクセス可能（MVPは限定URL）
- 利用端末：**MVP は自宅PC（Chrome / Edge）を動作保証対象**。iPad・スマホは閲覧可能だが動作保証対象外（将来ユーザー拡大時に対応）
- **生徒の個人情報は一切扱わない**

## 3. 用語定義

| 用語 | 定義 |
|---|---|
| カリキュラムパック | `publisher × subject × grade` 単位の共有データ（年間指導計画＋単元別授業内容）。複数校で再利用できる。 |
| 教員個人設定 | 学校名・年度・時間割・進度など、ユーザー固有のデータ |
| 年度 | 日本の学校年度（4月〜翌3月）。例：「2026年度」は 2026年4月〜2027年3月 |
| 第n週 | 始業式のある週を第1週とし、以降機械的に加算 |
| 本時 | ある単元内でのn時間目（例：「生き物をさがそう」の1/4時間目） |
| 週案 | 1週間の授業予定表。本システムの主要出力物 |
| Y案 | 理科専科独自1枚様式（全クラスを1枚に集約、管理職合意済み） |
| アーカイブ | 年度切替時に凍結保存された過去年度データ（読み取り専用） |
| 週先頭コマ確定 | 各クラスの今週最初のコマの単元・本時をユーザーが確定する操作。以降のコマは LessonMaster 順で自動表示 |
| 進度ヘルスチェック | 期待累計時数（年間配当×進行率）と実累計の差から進度ずれを判定する機能 |
| 月次集計 | 月別クラス×単元の実施時数マトリクス。教育委員会報告用 |

## 4. システム概要

### 4.1 システム構成
```
[ブラウザ (PC/iPad/スマホ)]
       │ HTTPS
       ▼
[Vercel (Next.js)]
  ├── 静的アセット
  ├── App Router 画面
  └── TS定数として格納されたカリキュラムパック
       │
       ▼（クライアント側）
[localStorage]
  └── 教員個人設定・進度・例外・メモ・アーカイブ
```

- サーバー側での永続化はMVPでは行わない
- 全データはブラウザの localStorage に保存
- バックアップは JSON エクスポート機能で対応

### 4.2 利用イメージ
1. 教員が朝、ホーム画面で今日の理科予定を確認
2. 週末に週案画面で翌週の予定を確認・メモ追記
3. 印刷画面から週案を A4横で出力 → 管理職に提出
4. 授業実施後、進度管理画面で完了時数を更新
5. 行事で時間割が変わる週は、時間割画面で例外を入力
6. 年度末に年度切り替えを実行 → 新年度は進度リセット、過去年度はアーカイブ

| F12 | マスタ確認 | 年間指導計画・授業内容の閲覧 |
| F13 | 年度切り替え | 現年度凍結 → 新年度初期化 |

| F16 | LINE通知用関数 | `getLessonsForDate` を公開（送信機能は未実装） |

### 5.2 機能詳細

#### F01 カリキュラムパック管理
- `publisher × subject × grade` をキーとする `CurriculumPack`
- 年間指導計画（`AnnualPlan`）と単元別授業内容（`LessonMaster`）を保持
- データはTS定数としてビルド時に組み込み
- `CurriculumPack.year` は発行年度を保持（MVP初期値 `2024`：令和5年検定済・令和6年発行）
- 教科書改訂時は新パックを追加、旧パックは破棄

#### F02 教員個人設定
- 学校名、教員名、**年度**、**始業式日**、使用パック選択
- 始業式日は第1週の起点として週番号算出に使用
- localStorage に年度プレフィックス付きで保存

#### F03 基本時間割マスタ
- 月〜土 × 時限 × クラスのマトリクス
- 各コマ：クラスコード、学年、組（**時刻は扱わない**：本システムは進度・時数管理が目的）
- MVPでは初期値TS定数＋画面から編集可能

#### F04 時間割例外
- `TimetableOverride` として日付・時限単位で保持
- 変更タイプ：`cancel`（休講）／`replace`（差し替え）／`add`（追加）
- 週案生成時にベース時間割とマージ

#### F05 進度管理
- クラスごとに `current_unit_name` と `completed_hours`、`total_completed_hours`
- 完了時数が当該単元の全時数を超えたら次単元へ繰り上げ
- 年度開始からの累計時数も保持（印刷の「実施累計」に使用）

#### F06 週案自動生成
- `generateWeeklyPlan(weekStart, setting, timetable, overrides, progress, packs, firstLessonConfirms?)`
- 月〜土 の6日分を生成
- 各クラスの週先頭コマはユーザー確定値があればそれを採用、なければ `ClassProgress` から推定
- 2コマ目以降：先頭コマの次から `LessonMaster` を順に割り当て、単元完了時は年間計画順で次単元へ

#### F07 週案表示
- 上部：第n週 / 期間 / クラス別集計（週実施・実施累計）・進度ヘルスバッジ
- **週先頭コマ確定エリア**：各クラスごとに単元・本時をプルダウンで確定
- 中段：曜日ごとのコマ一覧（先頭コマ以降は自動表示）
- 各コマ：時限・クラス・単元・本時・内容・メモ
- メモは inline 編集（blur で localStorage 保存）

#### F08 今日の授業一覧
- 本日の時限順カード表示
- クリックで週案画面へ

#### F09 週案印刷
- A4横、`@media print` 対応
- Y案：クラスコード＋単元名＋授業内容の3行で1セル
- 下段：クラス別の週実施／実施累計
- 検印欄は空欄（手書き）
- 週先頭コマ未確定時も `ClassProgress` 推定値で無警告印刷（画面上のみ `slate-400` で「未確定」バッジ表示、印刷面には出さない）

#### F10 月次集計
- `/monthly` 画面で月選択 → クラス×単元マトリクス表示
- `computeMonthSummary(year, month, ...)` で算出
- 月曜日が属する月で集計（週をまたぐ週は月曜日の月）
- 印刷対応（`/print/monthly`）、教育委員会提出用

#### F11 進度ヘルスチェック
- `checkProgressHealth(progress, annualPlan, currentWeekNo, totalWeeks)`
- 期待累計 = 年間配当時数 × (現在週 / 全週数)
- 差 ±3時間未満：ok（緑）、±3〜5時間：warn（amber）、±5時間以上：alert（rose）
- ホーム・週案・進度画面にバッジ表示

#### F12 マスタ確認
- 年間指導計画（学年×月）・授業内容（単元→本時一覧）
- MVPは閲覧のみ

#### F13 年度切り替え
- 設定画面からモーダルで実行
- 新年度番号、新しい始業式日を入力
- 時間割・パック選択の引き継ぎ可否を選択
- 現年度データを `ArchivedWeek[]` に変換して保存
- 新年度は進度リセット
- **週先頭コマ確定値（`first_lesson_confirms`）は新年度に引き継がない**（年度プレフィックス付きキーで自然分離、新年度は空から確定していく）

#### F14 アーカイブ閲覧
- 年度一覧から選択 → 週一覧 → 週案詳細
- 完全に読み取り専用
- 印刷も可能（当時の単元名・内容がそのまま保存されている）

#### F15 JSONエクスポート/インポート
- エクスポート：`FullSnapshot` 形式で全年度を1ファイル化、ダウンロード
- インポート：ファイル選択 → 確認ダイアログ → 全上書き

#### F16 LINE通知用関数
- `getLessonsForDate(date): WeeklyPlan[]`
- `formatLessonsForLine(lessons, date): string`
- MVPは関数のみ公開、送信基盤は将来実装

## 6. 非機能要件

### 6.1 UI/UX
- 日本語UI
- 学校現場に馴染む平易な表現
- 青・グレー系の落ち着いた配色
- タップ領域 ≥ 44px（将来の iPad 対応を見越した設計）
- **MVP の動作保証対象は PC（Chrome / Edge）のみ**。iPad・モバイルはレスポンシブ設計で閲覧可能にとどめる

### 6.2 パフォーマンス
- ローカル計算のみのため即応（画面遷移 < 500ms）
- 1週間の週案生成 < 100ms
- localStorage 読み書き < 50ms

### 6.3 保守性
- ロジック層（`lib/`）とUI層（`app/` `components/`）を分離
- 型定義は `types/index.ts` に集約
- データソース抽象層（`DataSource`）で永続化層を差し替え可能
- CSV ⇄ TS定数の相互変換を容易にするフラット構造

### 6.4 セキュリティ
- 生徒の個人情報は扱わない
- 公開URL運用可能（個人情報漏洩リスクなし）
- 将来、認証が必要になれば別途設計

### 6.5 可用性
- Vercel 無料枠で運用
- データは localStorage（クライアント側）
- JSONエクスポートでバックアップ可能
- サーバー障害時も過去データは閲覧可能（ブラウザキャッシュ）

### 6.6 運用性
- 年度切り替えはユーザー自身が設定画面から実行
- マスタ（カリキュラムパック）更新はコード変更＋再デプロイ
- 将来、CSVインポートや管理画面を追加予定

## 7. データ要件

### 7.1 データ構造の概要
詳細は `data_model.md` を参照。

**主要な型：**
- パック層：`CurriculumPack` / `AnnualPlan` / `LessonMaster`
- 個人設定層：`TeacherSetting` / `Timetable` / `TimetableOverride` / `ClassProgress` / `FirstLessonConfirm`
- 派生：`WeeklyPlan` / `WeekSummary` / `ClassTally` / `ProgressHealth` / `MonthlySummary` / `MonthlyClassTally`
- アーカイブ：`ArchivedWeek` / `ArchiveMetadata` / `FullSnapshot`

### 7.2 永続化
- マスタ：TS定数（ビルド時固定）
- 教員個人設定：localStorage（年度プレフィックス付きキー）
- バックアップ：JSONファイル（ユーザーのダウンロードフォルダ）

### 7.3 データ量見積
- 1年度あたり：週案約40週 × 6日 × 最大6コマ = 約1440レコード
- 進度：7クラス × 数値フィールド = ごく少量
- localStorage 上限（通常5MB）には余裕で収まる

## 9. 制約事項

### 9.1 技術制約
- Next.js（App Router）＋ TypeScript を使用
- ブラウザ環境（localStorage依存）
- MVPではサーバー側ストレージなし

### 9.2 運用制約
- 学校PCには導入しない（セキュリティ制約）
- 自宅PCからの利用が基本
- 端末間同期はJSONエクスポートで手動対応

### 9.3 スコープ制約
- 理科以外は対象外（MVP）
- 祝日・長期休業は未対応（画面上は平日扱い、手動で無視）
- 学校行事は `TimetableOverride` に手動入力
- **時刻（`start_time`/`end_time`）は扱わない**：本システムの目的は進度ずれとコマ数消化の管理であり、時間帯情報は不要
