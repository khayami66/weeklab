# requirements.md

## 1. プロジェクト概要
小学校教員向け、**週案作成支援Webアプリケーション**。
初期リリース（MVP）は **松戸市立梨香台小学校・理科専科・3/4年・啓林館** を対象とするが、設計は「**啓林館採用校であれば誰でも使える**」ことを前提とし、将来的に他教科・他出版社・他学年にも拡張できる構造にする。

## 2. 対象範囲

| 層 | MVP | 将来 |
|---|---|---|
| 学校 | 松戸市立梨香台小学校 | 啓林館採用校（松戸市内 → 全国） |
| 教科 | 理科 | 国算社 他 |
| 学年 | 3年・4年 | 全学年 |
| 教科書 | 啓林館 | 東京書籍・大日本図書・光村 など |
| ユーザー | 理科専科教員（本人） | 担任教員・他専科教員 |

## 3. 配信・運用形態
- **Webアプリ**（Next.js + Vercel）
- **利用端末は自宅PCのみ**（学校PCはセキュリティ制約で導入困難）
- 副次的に自宅内の iPad/スマホからも同一URLで確認可能
- **生徒の個人情報は一切扱わない**（成績管理は別システムで切り分け）

## 4. 解決したい課題
- 年間指導計画・時間割・進度・単元内容・週案様式を毎週手作業で突合している
- クラスごとに進度が異なり、次に何の単元の何時間目かを都度確認する手間
- 管理職提出用に、所定様式で週案を印刷する必要がある
- 行事等で時間割が流動的に変わるため、ベース時間割だけでは対応できない
- 年度が変わるたびに再構築が必要（進度リセット、始業式日変更、過去参照）

## 5. MVPで作る機能

1. **カリキュラムパック**の保持（啓林館×理科×3/4年の2パック）
2. **教員個人設定**（学校名・教員名・**年度**・**始業式日**・使用パックの選択）
3. **基本時間割マスタ**（理科専科用、月〜土、編集可）
4. **時間割の例外**（特定日・コマをプルダウンで差し替え）
5. **クラス別進度管理**（編集可、累計時数を保持、**進度ヘルスチェック**付き）
6. **週案自動生成**（指定週、ベース時間割＋例外を反映、**各クラスの週先頭コマをユーザー確定**→以降自動）
7. **週案表示画面**（曜日別・メモ編集可・週実施／実施累計表示・ヘルスバッジ）
8. **今日の授業一覧**（ホーム画面）
9. **週案印刷**（松戸市様式・理科専科版、月〜土の6行、週実施／実施累計）
10. **月次集計画面**（教育委員会報告用、クラス×単元マトリクス、印刷対応）
11. **マスタ確認**（年間指導計画・授業内容）
12. **年度切り替え**（現年度を凍結アーカイブ → 新年度を初期化）
13. **過去年度アーカイブ閲覧**（読み取り専用）
14. **JSONエクスポート/インポート**（バックアップ・復元）
15. **LINE通知用関数** `getLessonsForDate` の公開（送信機能は未実装）

## 6. MVPで作らないもの（将来拡張）

| 機能 | 方針 |
|---|---|
| 他教科・他出版社のカリキュラムパック | `data/curriculums/{publisher}/{subject}/grade{n}/` に追加 |
| PDF / Excel 出力 | ブラウザ印刷＋`@media print`で代替 |
| Googleスプレッドシート連携 | `DataSource` 抽象層で差し替え可能に |
| 学校行事の自動同期 | MVPは `TimetableOverride` に手動入力 |
| 教材研究メモ | `lesson_master` 拡張 or 別テーブル |
| 授業後の実施記録・反省 | `LessonRecord` を将来追加 |
| LINE通知（毎朝送信） | 関数のみMVP、送信基盤は後付け |
| 認証・マルチユーザー | MVPは自分用のため不要 |
| 成績管理 | 別システムとして切り分け、将来API連携 |
| DB / サーバー永続化 | MVPは localStorage |
| アーカイブの保持上限（3年等） | MVPは無期限、多ユーザー化時に導入 |

## 7. 将来拡張の方針（設計で確保する点）

- **データ層の二層構造**：カリキュラムパック（共有）／教員個人設定（ユーザー固有）
- **年度ベースのデータ管理**：`TeacherSetting.school_year` で年度を持ち、データキーに年度を含める
- **ロジックとUIの分離**：週案生成は `lib/` の純粋関数、UI・Cron・APIから共通利用
- **データソース抽象化**：`DataSource` インターフェース経由、段階的差し替え
- **CSV/スプレッドシート移行性**：マスタはフラット構造
- **印刷レイアウト**：`@media print` でMVPから対応、将来他自治体様式は `lib/print/templates/`
- **成績システム連携**：クラスコード・パックIDを共通キーに、API連携で疎結合

## 8. 技術構成
- Next.js（App Router）+ TypeScript + React + Tailwind CSS
- Vercel デプロイ（無料枠）
- ローカルデータ：TS定数 + localStorage
- 日本語UI、青・グレー系配色、レスポンシブ（PC/iPad）

## 9. 非機能要件
- 教員が迷わず使える操作フロー
- iPad / PC 両対応（タップ領域 ≥ 44px）
- ローカル計算のみのため即応
- 管理職に提出できる印刷品質
- データ消失リスクに対して JSONエクスポートで対応

## 10. 確定事項

| 項目 | 内容 |
|---|---|
| 配信形態 | Webアプリ（Vercel） |
| 授業内容の粒度 | 案A：1フィールドに統一 |
| 週案の様式 | Y案：理科専科独自1枚様式（管理職確認済み） |
| 時間割の変更 | ベース＋例外（プルダウン） |
| LINE通知の内容 | 週案内容に準じる（準備物なし） |
| 週の曜日範囲 | 月〜土の6行（常時表示） |
| 第1週の起点 | 始業式の週（教員設定、2026年度は 2026-04-06） |
| クラスコード形式 | `{学年}-{組}`（例：`3-1`） |
| 週の集計項目 | 週実施／実施累計（週案に表示） |
| 月実施時数 | 月次集計画面で別集計（月曜日が属する月基準） |
| 実施累計の起点 | 年度開始（4月）からの通算 |
| 時数の予定/実施 | 予定通り実施される前提で自動算出 |
| 進度管理方式 | 前週確定型（各クラスの週先頭コマをユーザー確定→以降自動） |
| 進度ヘルスチェック | 期待累計との差 ±3時間で amber、±5時間で rose |
| 時刻の扱い | **扱わない**（本システムは進度・時数管理が目的、時間帯情報は不要） |
| 年度切り替え | 手動操作、現年度は凍結アーカイブ |
| アーカイブ保持 | MVPは無期限 |
| 過去単元名の一貫性 | アーカイブは WeeklyPlan スナップショットで保存 |
| 生徒個人情報 | 扱わない |
| 成績管理 | 別システム（将来API連携） |

## 11. 仮置き事項（MVP段階で残るもの）

| 項目 | 仮置き内容 |
|---|---|
| 単元内の実施順 | `lesson_no` 昇順 |
| 完了時数の全時数超過 | 次単元（main優先）へ繰り上げ |
| 「関連単元」(sub) | 年間計画順に消化 |
| `lesson_master` 未整備単元 | 「未作成」バッジ表示、時限表示は続行 |
| 祝日・長期休業 | MVP未対応（平日として表示、手動で無視） |
